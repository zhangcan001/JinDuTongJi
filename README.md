# JinDuTongJi

监理工程师专用的工程进度管控台，用于查看楼栋、楼层、专业单位和施工内容的完成情况。界面采用太空舱中控大屏风格，支持 3D 沙盘、进度预警、Excel 导入和监理周报生成。

## 主要功能

- 中控大屏：展示总体完成率、关键节点、监理指令和项目覆盖范围。
- 3D 楼栋沙盘：按楼栋和楼层显示完成率，点击楼层查看具体施工情况。
- 楼层明细：展示单位、施工内容、进度、计划日期、实际日期和监理意见，并可直接编辑关联节点。
- 楼层联动：点击模型楼层后支持上一层/下一层切换，也可一键带入新增该楼层节点。
- 项目范围维护：支持维护楼栋层数、地下室、专业单位和施工内容，维护结果同步到表单、筛选器和 3D 模型。
- 节点编辑：节点台账支持新增、编辑、删除，能维护完成率、实际完成日期和监理意见。
- 整改闭环：整改项支持编辑、关联进度节点、记录复验意见和闭合日期。
- 专业筛选：支持按楼栋、专业单位、施工内容和状态筛选模型数据。
- 进度预警：自动识别滞后、临期、已完成和施工中节点。
- Excel 导入：通过进度表更新全局数据，并校验缺失字段或异常内容。
- 导入预览：Excel 解析后先展示新增、更新、异常、重复和字段差异，确认后再写入数据。
- 数据备份：支持导出完整 JSON 备份，并从备份恢复浏览器本地数据。
- 自动恢复点：导入、删除、恢复示例或恢复备份前自动保存最近 5 次状态，可一键回滚。
- 台账筛选：节点台账支持关键词、状态、楼栋、责任单位筛选和排序。
- 地下室剖面：单独展示地下室各系统进度。
- 监理周报：根据当前数据自动生成周报文本。

## 本地运行

这是一个静态前端项目，不需要安装依赖。可以直接打开 `index.html`，或用本地静态服务运行：

```powershell
npx serve .
```

如果当前端口是 `4173`，浏览器访问：

```text
http://127.0.0.1:4173/
```

## 在线部署

仓库已包含 GitHub Pages 工作流：`.github/workflows/pages.yml`。

推送到 `master` 后，GitHub Actions 会尝试部署静态页面。部署成功后，访问地址通常为：

```text
https://zhangcan001.github.io/JinDuTongJi/
```

如果首次使用 GitHub Pages，需要在仓库 Settings → Pages 中选择 GitHub Actions 作为部署来源。

## 技术说明

- `index.html`：页面结构。
- `styles.css`：太空舱中控大屏视觉样式。
- `js/data.js`：演示数据、兼容性工具和楼层示例数据生成。
- `js/dom.js`：页面元素引用和模型交互状态。
- `js/state-import.js`：本地存储、Excel 导入校验、导入应用、模板下载和 CSV 导出。
- `js/dashboard.js`：总览大屏、预警、偏差、单位排名和监理周报生成。
- `js/scope-model.js`：项目部位、Canvas 楼栋模型、楼层明细和地下室剖面。
- `js/records-chart.js`：节点、整改项和进度曲线渲染。
- `js/main.js`：事件绑定、全局渲染入口和页面初始化。
- `js/vendor/xlsx.full.min.js`：本地 Excel 解析依赖，避免运行时依赖外部 CDN。

数据默认保存在浏览器 `localStorage` 中，点击“恢复示例”可恢复演示数据。
