# JinDuTongJi

监理工程师专用的工程进度管控台，用于查看楼栋、楼层、专业单位和施工内容的完成情况。界面采用太空舱中控大屏风格，支持 3D 沙盘、进度预警、Excel 导入和监理周报生成。

## 主要功能

- 中控大屏：展示总体完成率、关键节点、监理指令和项目覆盖范围。
- 3D 楼栋沙盘：按楼栋和楼层显示完成率，点击楼层查看具体施工情况。
- 楼层明细：展示单位、施工内容、进度、计划日期、实际日期和监理意见，并可直接编辑关联节点。
- 楼层联动：点击模型楼层后支持上一层/下一层切换，也可一键带入新增该楼层节点。
- 项目范围维护：支持维护楼栋层数、地下室、专业单位和施工内容，维护结果同步到表单、筛选器和 3D 模型。
- 楼栋批量导入：支持按行录入楼栋和层数，快速初始化项目范围。
- 角色权限：支持管理员、项目经理、施工单位、监理和只读查看角色；只读角色不能修改数据。
- 单位权限：施工单位角色可按单位筛选可见数据，为后续真实账号隔离打基础。
- 全局搜索：可按楼栋、节点、单位、问题和意见快速筛选任务与整改项。
- 节点编辑：节点台账支持新增、编辑、删除，能维护完成率、实际完成日期和监理意见。
- 整改闭环：整改项支持编辑、关联进度节点、记录复验意见和闭合日期，并按未整改、整改中、待复验、已闭合分组展示。
- 专业筛选：支持按楼栋、专业单位、施工内容和状态筛选模型数据。
- 进度预警：自动识别滞后、临期、已完成和施工中节点。
- 多维分析：按楼栋、责任单位、专业和月份展示完成率、滞后与临期统计。
- 进度权重：支持按专业单位配置权重，总进度、楼栋进度和专项统计会按权重折算。
- Excel 导入：通过进度表更新全局数据，并校验缺失字段或异常内容；支持读取同一文件中的多个单位工作表。
- 多表模板：下载模板会按施工单位生成分表，每个单位表内按“每栋楼、每一层、每条施工内容”逐条列出。
- 导入预览：Excel 解析后先展示新增、更新、异常、重复和字段差异，确认后再写入数据。
- 导入策略：支持“更新已有并追加新增”“只更新已有”“只追加新增”，便于控制全量更新方式。
- 导入版本：确认导入后自动保存版本，可查看最近 10 次导入并恢复指定版本。
- 导入差异：可导出最近一次导入的新增、更新、跳过明细。
- 数据体检：自动检查重复节点、缺少计划时间、缺少楼栋/楼层、楼层超范围和进度状态冲突。
- 数据备份：支持导出完整 JSON 备份，并从备份恢复浏览器本地数据。
- IndexedDB 镜像：保存数据时同步写入浏览器 IndexedDB 最新快照，提升大数据量下的本地持久化能力。
- 自动恢复点：导入、删除、恢复示例或恢复备份前自动保存最近 5 次状态，可一键回滚。
- 操作记录：新增、编辑、删除、导入、恢复和角色切换会写入本地审计记录。
- 台账筛选：节点台账支持关键词、状态、楼栋、责任单位筛选和排序。
- 台账分页：节点台账按页渲染，避免大批量节点一次性生成 DOM 导致卡顿。
- 正式导出：支持导出节点台账、滞后清单、整改台账和监理周报。
- PDF 打印：支持把当前周报用浏览器打印为 PDF。
- 自动整改提醒：可根据滞后节点批量生成整改项。
- 地下室剖面：单独展示地下室各系统进度。
- 电梯专项看板：电梯单位数据单独按楼栋统计完成率。
- 数据字典：展示当前项目楼栋、单位和施工内容字典，减少手输错配。
- 监理周报：根据当前数据自动生成周报文本。

## 本地运行

项目支持两种本地运行方式。轻量使用时可以直接打开 `index.html` 或用静态服务运行；需要 SQLite 持久化、备份、日志和后端审计时，使用内置 Node 服务。

先安装 Node 依赖：

```powershell
npm install
```

静态模式：

```powershell
npx serve .
```

SQLite 后端模式：

```powershell
npm start
```

默认浏览器访问：

```text
http://127.0.0.1:4173/
```

如需开启后端写入鉴权，可设置管理员密码后启动：

```powershell
$env:JINDU_PASSWORD="your-password"
npm start
```

如需固定会话 token 或在 HTTPS 反向代理后启用 `Secure` Cookie，可设置：

```powershell
$env:JINDU_TOKEN="replace-with-a-long-random-token"
$env:JINDU_SECURE_COOKIE="1"
```

## 本地检查

项目包含轻量 Node 测试和 UI 冒烟测试，用于覆盖进度状态、整改流转、延期原因分类、CSV 转义、导入校验、状态迁移、权限过滤、IndexedDB 镜像行为、Service Worker 缓存策略、后端鉴权/路径防护和页面基础渲染：

```powershell

```

```powershell

```

## 在线部署

仓库已包含 GitHub Pages 工作流：`.github/workflows/pages.yml`。

推送到 `master` 后，GitHub Actions 会先安装依赖并运行测试，测试通过后再部署静态页面。部署成功后，访问地址通常为：

```text
https://zhangcan001.github.io/JinDuTongJi/
```

如果首次使用 GitHub Pages，需要在仓库 Settings → Pages 中选择 GitHub Actions 作为部署来源。

## Excel 导入字段

模板表头固定为：

- 楼栋
- 楼层
- 专业
- 施工内容
- 计划完成时间
- 实际完成情况

“实际完成情况”为下拉选择：未开始、已完成、施工中。导入统计时，未开始按 0% 计算，已完成按 100% 计算，施工中按 50% 计算。页面内提供模板下载按钮，会按当前项目的施工单位生成多工作表 Excel。每个施工单位只填写自己对应的工作表，上传时系统会自动读取所有单位分表。

电梯单位模板单独使用以下表头：楼栋、专业、电梯数量、已安装数量、完成百分比。其中完成百分比由 Excel 公式自动按“已安装数量 / 电梯数量 × 100”计算。

## 技术说明

- `index.html`：页面结构。
- `styles.css`：太空舱中控大屏视觉样式。
- `js/data.js`：演示数据、兼容性工具和楼层示例数据生成。
- `js/business-rules.js`：进度状态、整改流转、延期原因分类、日期和 CSV 转义等可测试业务规则。
- `js/project-helpers.js`：项目、楼栋、楼层、单位、节点去重键和导入范围补齐等共享辅助函数。
- `js/dom.js`：页面元素引用和模型交互状态。
- `js/validation-schema.js`：任务、整改和导入数据的前端结构校验。
- `js/error-reporting.js`：前端异常捕获，并在后端模式下写入审计日志。
- `js/ui-dialogs.js`：统一 Toast 提醒和确认弹窗。
- `js/storage-audit.js`：本地存储、IndexedDB 镜像、审计记录、恢复点和完整备份恢复。
- `js/import-loader.js`：按需加载 Excel/CSV 导入相关脚本，减少首屏脚本体积。
- `js/state-import.js`：权限、项目状态迁移、计划基线、数据体检和 CSV/周报导出。
- `js/import-excel.js`：Excel 导入校验、预览、确认导入、待复核导入、导入版本和模板下载。
- `js/dashboard.js`：总览大屏、预警、偏差、单位排名、多维分析和监理周报生成。
- `js/scope-model.js`：项目部位、Canvas 楼栋模型、楼层明细和地下室剖面。
- `js/records-chart.js`：节点、整改项和进度曲线渲染。
- `js/main.js`：事件绑定、全局渲染入口和页面初始化。
- `js/vendor/xlsx.full.min.js`：本地 Excel 解析依赖，避免运行时依赖外部 CDN。

## 维护说明

- 数据仍以浏览器 `localStorage` 为首屏启动来源，并会延迟镜像到 IndexedDB 的 `JinDuTongJiDB/snapshots/latest`。如果 `localStorage` 缺失，页面启动后会尝试从 IndexedDB 最新快照恢复。
- `js/import-excel.js` 和 `js/import-file-reader.js` 由 `js/import-loader.js` 在导入、粘贴或下载模板时按需加载。
- `js/vendor/xlsx.full.min.js` 是本地 Excel 解析依赖；升级时请替换该文件并完成 `npm test` 和一次浏览器导入预览验证。
- Excel 导入单文件限制为 8MB，解析后最多 10000 行；超过限制建议按施工单位或楼栋拆分导入。
- 大文件后续拆分方向：`import-excel.js` 可按解析、校验、预览、应用导入、模板导出拆分；`scope-model.js` 可按范围维护、楼栋模型、楼层详情拆分。
- 渲染后续优化方向：将当前全量 `render()` 调用逐步收敛为任务、范围、导入、配置等局部刷新函数。
- 后端会同步维护 `projects`、`tasks`、`issues` 等结构化索引表，并提供分页查询接口；完整 JSON 快照仍作为恢复和版本回滚来源。

点击“恢复示例”可恢复演示数据。
